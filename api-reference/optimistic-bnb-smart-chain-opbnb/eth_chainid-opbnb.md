# eth\_chainId - opBNB

This method returns the chain ID of the connected network. For opBNB mainnet, the result is `0xcc` (204 in decimal); for testnet, `0x15eb` (5611). Chain ID is used in EIP-155 signed transactions to prevent replay attacks across chains.

## Parameters

* None

use reqwest::Client;
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = Client::new();

    let payload = json!({
        "jsonrpc": "2.0",
        "method": "eth_chainId",
        "params": [],
        "id": "getblock.io"
});

    let response = client
        .post("https://go.getblock.io/<ACCESS-TOKEN>/")
        .header("Content-Type", "application/json")
        .json(&payload)
        .send()
        .await?;

    let result: serde_json::Value = response.json().await?;
    println!("{:#?}", result);

    Ok(())
}

## Response Parameters

| Field  | Type   | Description                                                             |
| ------ | ------ | ----------------------------------------------------------------------- |
| result | string | Chain ID in hexadecimal (`0xcc` = 204 mainnet, `0x15eb` = 5611 testnet) |

## Use Cases

* Constructing EIP-155 signed transactions
* Validating that a wallet is connected to the expected network
* Multi-chain dApps that branch logic by chain ID
* Configuring transaction signers in Ethers.js / Viem

## Error Handling

| Status Code | Error Message     | Cause                                       |
| ----------- | ----------------- | ------------------------------------------- |
| 403         | Forbidden         | Missing or invalid `<ACCESS-TOKEN>`         |
| -32602      | Invalid params    | Request parameters are missing or malformed |
| -32601      | Method not found  | The method is not supported by this node    |
| 429         | Too Many Requests | Rate limit exceeded for your plan           |
