# eth\_syncing - Worldchain

Returns an object with data about the sync status or false if not syncing on the World Chain network.

### Parameters

* None

### Request

{% tabs %}
{% tab title="curl" %}
```bash
curl --location --request POST 'https://go.getblock.io/<ACCESS-TOKEN>/' \
--header 'Content-Type: application/json' \
--data-raw '{
    "jsonrpc": "2.0",
    "method": "eth_syncing",
    "params": [],
    "id": "getblock.io"
}'
```
{% endtab %}

```javascript
    .catch(error => console.log(error));
```

### Response

```json
{
    "jsonrpc": "2.0",
    "id": "getblock.io",
    "result": false
}
```

### Response Parameters

| Field  | Type    | Description |
| ------ | ------- | ----------- |
| result | boolean | object      |

### Use Case

The `eth_syncing` method on World Chain is typically used for:

* Sync status checking
* Node health monitoring
* Connection verification
* Operational monitoring

### Error Handling

| Status Code | Error Message | Cause                           |
| ----------- | ------------- | ------------------------------- |
| 403         | Forbidden     | Missing or invalid ACCESS-TOKEN |
