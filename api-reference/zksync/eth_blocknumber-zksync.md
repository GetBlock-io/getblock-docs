# eth\_blockNumber - zkSync

Returns the number of the most recent L2 block on zkSync Era. With \~1 second block times, this is the standard way to poll for new blocks. Note: L2 blocks are grouped into L1 batches — use `zks_L1BatchNumber` for batch-level tracking.

## Parameters

* None

use reqwest::Client;
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = Client::new();

    let payload = json!({
        "jsonrpc": "2.0",
        "method": "eth_blockNumber",
        "params": [],
        "id": "getblock.io"
});

    let response = client
        .post("https://go.getblock.io/<ACCESS-TOKEN>/")
        .header("Content-Type", "application/json")
        .json(&payload)
        .send()
        .await?;

    let result: serde_json::Value = response.json().await?;
    println!("{:#?}", result);

    Ok(())
}

## Response Parameters

| Field   | Type   | Description                           |
| ------- | ------ | ------------------------------------- |
| jsonrpc | string | JSON-RPC protocol version (`2.0`)     |
| id      | string | Request identifier echoed back        |
| result  | string | Latest L2 block number in hexadecimal |

## Use Cases

* Polling for new L2 blocks in indexers and dApps
* Determining the current chain tip before making other calls
* Health-checking the RPC endpoint

## Error Handling

| Status Code | Error Message     | Cause                                       |
| ----------- | ----------------- | ------------------------------------------- |
| 403         | Forbidden         | Missing or invalid `<ACCESS-TOKEN>`         |
| -32602      | Invalid params    | Request parameters are missing or malformed |
| 429         | Too Many Requests | Rate limit exceeded for your plan           |

result = zk_web3.zksync._zks_endpoints if 'eth_blockNumber'.startswith('zks_') else None
