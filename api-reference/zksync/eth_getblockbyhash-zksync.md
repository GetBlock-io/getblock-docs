# eth\_getblockbyhash zksync

Returns information about an L2 block by its hash. Same response schema as `eth_getBlockByNumber` including the zkSync-specific `l1BatchNumber` field.

## Parameters

| Parameter        | Type    | Required | Description                                                          |
| ---------------- | ------- | -------- | -------------------------------------------------------------------- |
| blockHash        | string  | Yes      | 32-byte hash of the block                                            |
| fullTransactions | boolean | Yes      | If `true`, returns full transaction objects; if `false`, only hashes |

## Response Parameters

| Field                | Type           | Description                                                                  |
| -------------------- | -------------- | ---------------------------------------------------------------------------- |
| result               | object \| null | Block object (same schema as `eth_getBlockByNumber`), or `null` if not found |
| result.number        | string         | L2 block number (hex)                                                        |
| result.hash          | string         | L2 block hash                                                                |
| result.l1BatchNumber | string         | L1 batch number containing this L2 block                                     |

## Use Cases

* Resolving a block from a known hash (e.g. from a receipt)
* Indexers that walk by hash rather than height
* Cross-referencing zkSync L2 blocks with their L1 batches

## Error Handling

| Status Code | Error Message     | Cause                                                |
| ----------- | ----------------- | ---------------------------------------------------- |
| 403         | Forbidden         | Missing or invalid `<ACCESS-TOKEN>`                  |
| -32602      | Invalid params    | Request parameters are missing or malformed          |
| -32601      | Method not found  | Method does not exist or is not enabled on this node |
| 429         | Too Many Requests | Rate limit exceeded for your plan                    |
